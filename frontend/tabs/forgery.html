How It Works

Three forensic computation steps running locally on CPU

01

Multi-Scale Conversion

Ingested PDFs are converted to high-resolution images. OpenCV generates edge contour mappings, binarized matrices, and normalizes colorspaces.

02

Forensic Check Routine

Parallel CV algorithms scan for duplicate template segments, stroke inconsistencies, erasure blur coefficients, and horizontal noise mismatches.

03

Overlay & Report Output

Aggregates findings, renders colored bounding boxes, highlights anomalies in preview layers, and issues local validation results and structured JSON logs.

100% Confidential On-Premises Analysis

All file parsing, edge calculations, and pixel-level checks execute locally on your host environment. No document bytes are transmitted externally, no files are stored permanently, and all active memory buffers are purged immediately after job completion.

Service Limitations

25MB File Limit

Maximum single file upload size is capped at 25MB. Supported formats: PDF, JPG, PNG, BMP, TIFF, WebP.

Digital Alterations Only

Focuses on digital anomalies (copy-paste, erasures, metadata edits). Physical paper forgery (e.g., ink chemistry) is not detectable from scans.

Not Fully Accurate

Very low resolution or heavily compressed documents can produce noise that might trigger false positives or miss minor digital edits.

 Usage Examples


        

            

                
1. Upload a document

                
Send as multipart/form-data. Returns a job_id.

                

                    
curl -X POST "https://dpi.tanuh.ai/forgensic/jobs" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -F "file=@/path/to/document.pdf" \
  -F "ocr_enabled=true"

                    
                

            

            

                
2. Poll job status

                
Poll until status is "complete" or "error".

                

                    
curl -H "Authorization: Bearer YOUR_TOKEN" \
  "https://dpi.tanuh.ai/forgensic/jobs/JOB_ID"

                    
                

            

            

                
3. Fetch results + preview

                
Results include preview URLs; fetch them with the same token.

                

                    
curl -H "Authorization: Bearer YOUR_TOKEN" \
  "https://dpi.tanuh.ai/forgensic/jobs/JOB_ID/results"

                    
                

            

        


        

            
Python: End-to-end flow

            

                
import time, requests

BASE = "https://dpi.tanuh.ai/forgensic"
headers = {"Authorization": "Bearer YOUR_TOKEN"}

with open("document.pdf", "rb") as f:
    r = requests.post(
        f"{BASE}/jobs",
        files={"file": f},
        data={"ocr_enabled": "true"},
        headers=headers,
        timeout=60,
    )
job_id = r.json()["job_id"]

for _ in range(30):
    s = requests.get(f"{BASE}/jobs/{job_id}", headers=headers, timeout=30).json()
    if s["status"] in ("complete", "error"):
        break
    time.sleep(2)

results = requests.get(f"{BASE}/jobs/{job_id}/results", headers=headers, timeout=30).json()
print(results)

                Uploads are written to a temporary job folder while processing and then cleaned up. Annotated preview files are cached in memory for the job and cleared on server restart. Jobs are also purged after the TTL window configured on the server.
