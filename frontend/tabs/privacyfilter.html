Python

    

        
import requests

resp = requests.post(
    "https://dpi-dev.tanuh.ai/privacy-filter/api/demo-token",
    json={"name": "Your Name", "email": "you@example.com"},
)
token = resp.json()["access_token"]

with open("document.pdf", "rb") as f:
    r = requests.post(
        "https://dpi-dev.tanuh.ai/privacy-filter/api/redact",
        headers={"Authorization": f"Bearer {token}"},
        files={"file": ("document.pdf", f, "application/pdf")},
        timeout=600,
    )
print(r.json()["entity_counts"])

Privacy Filter

                
CPU-based de-identification service that detects and redacts Personally Identifiable Information (PII) and Protected Health Information (PHI) from medical images, DICOM/NIfTI scans, and documents — removing both metadata and burned-in pixel text while preserving all clinical content.

            


            
Key Capabilities

            

                
FeatureDescription

                
                    
Metadata De-identificationScrubs DICOM tags, EXIF, and NIfTI headers; regenerates DICOM UIDs

                    
Burned-in Text RemovalShape-based connected-component detection + Tesseract OCR find and mask patient text overlaid on the image

                    
Modality-Aware ScanningFundus, OCT, ultrasound, CT, and MRI layouts handled with format-specific overlay regions, so clinical content is never touched

                    
Multi-Format SupportDICOM, NIfTI, images (PNG/JPG/TIFF/BMP), and PDF

                    
On-Premises ProcessingAll data stays on the server — no external API calls, fully DPDP Act compliant

                    
Fast ProcessingTypically 10–30 seconds per file on CPU

                
            

        


        
        

            
How to Use

            
Follow these three steps to de-identify a document.


            

                

                    
1. Upload File

                    
Navigate to the Privacy Filter tab and click "Choose document to redact...". Upload a DICOM, NIfTI, image, or PDF file. Maximum file size is 25 MB.

                

                

                    
2. Redact File

                    
Click "Redact Document". The system clears metadata tags and detects burned-in text using CV + OCR. Processing takes 10–30 seconds.

                

                

                    
3. Review & Download

                    
Review detected entities in the results panel. Burned-in text is masked with opaque boxes and metadata values are replaced with REDACTED. Download the original and de-identified versions.

                

            


            

                Accuracy Notice
                Detections may be incomplete or incorrect. Please review every output before use in production. The system prioritizes recall (catching more PHI) over precision.
            

        


        
        

            
Detection Pipeline

            
The Privacy Filter uses a layered, CPU-only pipeline that removes patient information from both the file's metadata and the image pixels, without altering any clinical content.


            
Layer 1 — Metadata De-identification

            
DICOM tags (patient name, ID, birth date, sex, institution, physician, device, …) are scrubbed and study/series/SOP UIDs regenerated; EXIF and NIfTI header fields are cleared. Pixel and voxel data are left untouched.


            
Layer 2 — Burned-in Text Detection

            
A shape-based connected-component scan locates character-like regions where overlays sit, without reading them. The scan is modality-aware: ophthalmic images (fundus, OCT) use a narrow band inside the black border, while ultrasound / CT / MRI use a full-width header band — so measurements, calipers, and clinical annotations are never masked.


            
Layer 3 — OCR + Rule-Based Classification

            
Tesseract OCR reads the detected regions and deterministic rules classify the text as patient information — names, dates, identifiers, and ages. Confirmed regions are masked with opaque boxes; the redaction is irreversible and leaves surrounding pixels unchanged.


            
Processing Flow

            

                
1. Load & Format Detection
File type identified and routed to the matching loader (DICOM, NIfTI, image, PDF).

                
2. Metadata Scrub
Header/tag PHI removed; DICOM UIDs regenerated.

                
3. Text-Region Detection
Connected-component scan locates burned-in text in modality-specific regions.

                
4. OCR & Rule Scan
Tesseract reads regions; rules flag names, dates, and IDs.

                
5. Masking
Detected regions blacked out; clinical content preserved.

                
6. Validation & Output
Redacted image re-checked for residual text; de-identified file written in the original format.

Privacy & Data Handling

            
The Privacy Filter is designed with data minimization as a core principle.


            
    
                
  • On-premises only — all processing happens locally, no data leaves the server
    • 
                
  • No storage — uploaded files are deleted immediately after processing
    • 
                
  • No logging — no document content is logged or persisted to disk
    • 
                
  • Anonymous tracking — visitor IPs stored as one-way hashes only
    • 
                
  • DPDP Act compliant — meets India's data protection requirements for data minimization
    • 
            


            

                No External API Calls
                Unlike cloud-based solutions, the Privacy Filter runs entirely on-premises. Your files never leave your infrastructure. All detection runs locally on CPU.
