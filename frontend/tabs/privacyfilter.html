Privacy Filter (Anonymization)

Privacy Filter (Anonymization) is an automated, interactive, and on-premises-ready service for anonymizing Personal Identifiable Information (PII) and Protected Health Information (PHI) from clinical documents, medical images, and DICOM/NIfTI scans. Compliant with the DPDP Act and HIPAA Safe Harbor, it operates as a non-persistent service, processing data in real time and purging it automatically upon completion without retaining any information.

Key Capabilities

Metadata Scrubbing

Deep header parsing for medical scans: automatically scrubs nested DICOM tags, EXIF details, and NIfTI properties, while safely regenerating unique UIDs.

Burned-in Text Removal

Utilizes advanced connected-component analysis and Tesseract OCR to target, detect, and mask patient credentials overlaid directly on pixel scans.

DPDP & HIPAA Compliance

Identifies and masks HIPAA Safe Harbor identifiers (names, dates, emails, phone numbers, identifiers, and location tags) within text and documents.

Interactive Revision Editor

A built-in manual correction editor enables clicking or drawing bounding boxes to instantly black out and mask any missed entities.

How It Works

Four layered processing phases executing fully on local CPU

01

Data Parsing & Extraction

Accepts PDFs, scans, DICOM, or NIfTI images. Direct text parsing or OCR processes content, drawing absolute coordinate boundaries for each term.

02

PII Detection Pipeline

Executes advanced regex pattern mapping, lookup lists, and layout-aware heuristics to match and isolate target sensitive terms in real-time.

03

Masking & Fluid Redaction

Constructs an overlay mask on coordinate values, rendering vector black boxes over raw characters or pixel arrays, and yields sanitised PDF, PNG, or JSON files.

04

Interactive

Designed for expert review, the built in editor allows users to redact missing PII/PHI or remove incorrect redactions with instant visual re-rendering.

Plausible 100% Confidential On-Premises Execution

Deploy the Privacy Filter service locally with a simple CPU-based installation. No external API calls, no cloud storage, and all processing occurs securely in memory. Designed to support compliance with the Indian DPDP Act for privacy-preserving healthcare applications.

Service Limitations

25MB File Limit

Maximum single file upload size is capped at 25MB to ensure stable memory consumption during in-memory processing.

Manual Review Required

OCR and heuristic matching may fail on low-contrast scans or non-standard fonts. Use the built-in manual correction editor to review and finalize redactions.

Long PDF Processing Time

Multi-page PDFs requiring deep OCR text extraction may take longer (up to 30 seconds) to render completely on CPU resources.

 Quick Start


        

            
curl

            

                
curl -X POST https://dpi.tanuh.ai/privacy-filter/api/redact \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -F "file=@/path/to/document.pdf"

                
            

        


        

            
Python

            

                
import requests

with open("document.pdf", "rb") as f:
    r = requests.post(
        "https://dpi.tanuh.ai/privacy-filter/api/redact",
        headers={"Authorization": "Bearer YOUR_TOKEN"},
        files={"file": ("document.pdf", f, "application/pdf")},
        timeout=600,
    )
print(r.json()["entity_counts"])

Setup Steps (All Platforms)


            

                
1. Download Executable

                

                    Download the pf-redact binary for your OS from the links above.
                

            


            

                
2. Unzip & Make Executable (Linux / macOS)

                

                    Unzip the archive, then: chmod +x pf-redact*
                

            


            

                
3. Verify Setup

                

                    ./pf-redact* check
                

                

                    Confirms Tesseract OCR and all dependencies are available. No model download required - detection runs entirely on CPU.
                

            


            

                
4. Redact a Document

                

                    ./pf-redact* document.png redacted.png --backend tesseract --json report.json
                

                

                    Outputs a redacted file + JSON report with PHI count, overlay count, validation status, and risk score. Supports DICOM, NIfTI, images (PNG/JPG/TIFF/BMP), and PDF.
                

            


            

                

                     Privacy: All data stays on your machine. No models are downloaded - detection runs locally on CPU. Zero internet or cloud API calls are made.
