Key Capabilities

NHCX Schema Compiler

Fully compiles unstructured medical policy conditions, limits, networks, and copays directly into NHCX Schema StructureDefinitions.

OCR Waterfall Engine

Employs 6 OCR engines including PyPDF, Chandra2 VLM layouts, Docling table solvers, and Tesseract defaults to handle scan artifacts flawlessly.

Structured FHIR Output

Constructs ready-to-transmit FHIR R4 InsurancePlan, Organization, Patient, and Condition resources tied together in a single interoperable Bundle.

On-Premises Runtime

Run locally using standalone zero-dependency packages for Windows, Mac, or Linux, keeping sensitive medical policy information completely secure offline.

How It Works

Three conversion steps running securely on the platform

01

Ingestion & Verification

PDF policies are OCRed, categorized, and fed to a multi-tier document validator. Unrelated files are automatically identified and flagged.

02

LangGraph Extraction

A dependency-ordered LangGraph workflow pulls details (insurers, co-payments, room ceilings, deductibles) by referencing official NHCX rulesets.

03

FHIR Assembly & Validation

Constructs cross-references, validates codes against ICD-10/SNOMED CT, embeds the original PDF as base64, and compiles a clean, validated NHCX Bundle.

100% Confidential On-Premises Analysis

All extraction processes, OCR computation layers, and FHIR mappings compile in memory locally. No data bytes are logged or transmitted externally, maintaining compliance with national healthcare architecture guidelines and privacy protocols.

Setup Steps (All Platforms)

1. Install Ollama

Download from ollama.com/download
Linux: curl -fsSL https://ollama.com/install.sh | sh

2. Pull the LLM Model (~16 GB download)

ollama pull gemma4:26b
Requires NVIDIA GPU with 16 GB+ VRAM, or Apple Silicon with 16 GB+ unified memory.

3. Unzip & Make Executable

Unzip the downloaded file. On Linux/Mac run: chmod +x nhcx-extract*
Linux requires sudo apt install libfuse2 if not already installed.

4. Run NHCX Insurance Extraction

./nhcx-extract nhcx policy.pdf -o result.json -m gemma4:26b

Use check first to verify setup: ./nhcx-extract check -m gemma4:26b

Privacy: All data stays on your machine. Zero cloud API calls. The output JSON contains a complete FHIR R4 Bundle.

Insurance Policy API

Developer API for converting insurance policy PDFs into NHCX-compliant FHIR InsurancePlanBundles.

Authentication: Uses JWT bearer tokens. Request a 1-day demo token via the form below, then pass it as Authorization: Bearer <token>.
UI Uploads: When using the Insurance Policy tab, tokens are fetched automatically.

 Quick Start


        

            
curl

            

                
TOKEN=$(curl -s -X POST https://dpi-dev.tanuh.ai/pdf2nhcx/api/token \
  -H "Content-Type: application/json" \
  -d '{"name":"Your Name","email":"you@example.com"}' \
  | python3 -c "import sys,json; print(json.load(sys.stdin)['access_token'])")

curl -X POST https://dpi-dev.tanuh.ai/pdf2nhcx/submit \
  -H "Authorization: Bearer $TOKEN" \
  -F "file=@/path/to/insurance_policy.pdf"

                
            

        


        

            
Python

            

                
import requests

resp = requests.post(
    "https://dpi-dev.tanuh.ai/pdf2nhcx/api/token",
    json={"name": "Your Name", "email": "you@example.com"},
)
token = resp.json()["access_token"]

with open("insurance_policy.pdf", "rb") as f:
    r = requests.post(
        "https://dpi-dev.tanuh.ai/pdf2nhcx/submit",
        headers={"Authorization": f"Bearer {token}"},
        files={"file": ("policy.pdf", f, "application/pdf")},
        timeout=600,
    )
print(r.json())
