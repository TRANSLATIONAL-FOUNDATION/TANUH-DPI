Document Forgery Detection Service

Forgensic - Classical CV-Based Tampering Detection Complete Technical & User Documentation

1. Executive Summary

The Forgensic service is a production-ready document forgery detection platform built entirely on classical computer vision methods - no neural networks, no GPU requirements, 100% explainable results. It detects 10 categories of document tampering (C1-C10) including copy-paste, overwriting, added content, erasure, document merging, watermark removal, irregular spacing, and AI-generated content.

The service is designed for the high-throughput, low-latency demands of healthcare document verification in the AB-PMJAY ecosystem. Every detection traces back to a clear mathematical equation, making results legally defensible and auditable.

It runs as a FastAPI application on port 8004 with asynchronous processing via Celery workers on the dedicated 'forgensic' queue. Processing time: 1-3 seconds per page on standard CPU hardware.

3. Architecture

3.1 System Architecture

The service follows a producer-consumer pattern:

  • FastAPI server (port 8004) accepts uploads, validates files, creates job entries in Redis, and enqueues Celery tasks
  • Celery workers (4 replicas x concurrency 2 = 8 slots) consume from the 'forgensic' queue and run the heavy CV pipeline
  • Redis stores job state, results, and file maps with 1-hour TTL
  • Shared disk volume (forgensic_data/) stores input files, rendered pages, annotated previews, and export files

3.2 Detection Pipeline Per Page

  1. Page Quality Assessment: Compute Laplacian blur score, edge density, contrast ratio, and text density to characterize document quality.
  2. C1 Copy-Move Detection: Extract ORB keypoints (5000 features, scale_factor=1.2, 8 levels). Self-match via BFMatcher with Lowe's ratio test (0.75). Cluster matches by spatial shift vector (bin_size=14px). Flag clusters with >= 4 match pairs.
  3. C2 Overwriting Detection: Extract Canny edges (40-140). Compute local texture maps (residual, variance, gradient) per text line. Z-score threshold on edge/stroke density. Flag components where >= 2 signals exceed thresholds simultaneously.
  4. C3 Added Content Detection: Extract connected components. Classify by shape: stamps (circularity > 0.5, fill > 0.3), signatures (aspect > 3.0, fill < 0.2), regular text (fill > 0.55). Filter by area and aspect ratio.
  5. C4 Erasure Detection: Extract OCR token boxes. Compute inter-token gaps in text lines. Z-score large gaps as outliers. Detect smooth regions (low variance/gradient/residual ratios vs. surrounding ring).
  6. C5 Merge Detection: Compute row/column density heatmaps. Find horizontal/vertical discontinuities. Compare header vs. body noise fingerprint vectors.
  7. C6 Watermark Removal: Extract high-variance candidate regions. Detect diagonal lines via Hough transform. Check periodicity via FFT of band density.
  8. C7 Irregular Spacing: Extract OCR token boxes per text line. Compute median and MAD (median absolute deviation) of inter-token gaps. Z-score anomalous gaps.
  9. C8 AI-Generated Document: Compute FFT spectrum. Measure flatness, peak_ratio, highfreq_ratio. If page-wide coverage > 50% with AI signatures, flag as C8.
  10. C9 AI-Generated Edits: Per-candidate region: compute profile (edge density, gradient, residual, variance, stroke), z-score against page background, analyze FFT spectrum.
  11. Post-processing: Clip regions to image bounds, filter tiny regions (< 200px area), apply NPV focus filter if preset is 'npv_focus', validate predictions, render annotated preview image.

4. API Reference

4.1 Authentication

POST /forgensic/api/token with {name, email} to get a 1-day HS256 JWT signed with FORGENSIC_SECRET_KEY. Include as Authorization: Bearer . Auth bypass: set FORGENSIC_AUTH_ENABLED=false.

4.3 Job Lifecycle

  1. POST /forgensic/jobs - Upload file (multipart form-data), optional ocr_enabled parameter. Returns {job_id, status: 'queued'}.
  2. GET /forgensic/jobs/{job_id} - Poll status. Returns {status: 'queued'|'processing'|'complete'|'error', progress: 0.0-1.0, message: '...'}.
  3. GET /forgensic/jobs/{job_id}/results - When status='complete', returns full JobResultResponse with pages, category_summary, findings_summary, export_urls.
  4. GET /forgensic/jobs/{job_id}/files/{name} - Download specific output files (annotated previews, submission.json, submission_preview.xlsx, YAML annotations).

4.4 Result Format

The JobResultResponse contains:

  • pages: Array of page objects, each with page_number, image_url, preview_url, categories (list of C1-C10), regions (bounding boxes with category, type, stretch_factor), and quality notes
  • category_summary: {C1: count, C2: count, ...} across all pages
  • findings_summary: Human-readable text + structured findings with page, category, snippet (OCR text), location (e.g., 'top-center'), bounding box, and summary text
  • export_urls: Links to submission.json, submission_preview.xlsx, and per-page YAML annotations
  • inference_seconds: Total processing time

6.2 Zero-Persistence Privacy

Document data is temporarily stored on the shared volume and automatically purged after JOB_TTL_SECONDS (default: 1 hour). No data is permanently stored. This ensures compliance with healthcare data privacy requirements.

7. Security

  • All processing occurs locally on CPU - no external AI services or cloud inference
  • No document data leaves the server
  • Results auto-expire after 1 hour
  • 100% explainable: every detection traces to a mathematical equation
  • JWT authentication on all job endpoints

9. User Guide

9.1 Browser Usage

  1. Navigate to the DPI Platform website
  2. Click the 'Forgery Detection' tab
  3. Upload a PDF or image file
  4. Watch the progress bar as the CV pipeline processes each page
  5. View the tampering verdict: Tampered (True/False/Unknown)
  6. Explore the findings panel with category-coded, collapsible findings
  7. View the annotated document preview with bounding box overlays
  8. Click on individual regions to zoom in via the crop modal
  9. Copy findings text or download the annotated preview / export files

9.2 API Usage (curl)

# Get token

curl -X POST http://:8004/forgensic/api/token -H "Content-Type: application/json" -d '{"name": "Name", "email": "email@example.com"}'

# Submit document for analysis

curl -X POST http://:8004/forgensic/jobs -H "Authorization: Bearer " -F "file=@document.pdf"

# Poll status

curl http://:8004/forgensic/jobs/ -H "Authorization: Bearer "

# Get results

curl http://:8004/forgensic/jobs//results -H "Authorization: Bearer "

# Download annotated preview

curl -o preview.jpg http://:8004/forgensic/jobs//files/preview_page_1.jpg -H "Authorization: Bearer "

11. Offline Desktop Application

A standalone desktop GUI (forgensic_gui.py) is available for offline usage using CustomTkinter. It provides the same CV detection pipeline without requiring a server, Redis, or network connectivity. Results are saved locally as annotated images and findings text files.

12. Known Limitations & Future Enhancements

  • High false positive rate on very noisy/low-quality scans (use 'strict' preset)
  • OCR-dependent detectors (C4, C7) require Tesseract to be installed
  • No learning/adaptation - all thresholds are static per preset
  • No rate limiting on API endpoints
  • C8/C9 AI detection may not catch latest generation AI models
  • Planned: adaptive thresholding, batch processing, ensemble scoring, ML-augmented detection
